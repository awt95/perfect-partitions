Here is the pseudo code for the calculator:
  • Convert target size into mebibytes (MiB)
    • For TB: result = target size * 1024 * 1024
    • For GB: result = target size * 1024
    • For MB: result = target size * 1.024, round up to nearest integer
  • Add one MiB to compensate for additional overhead
    n.b. Actual overhead size is apparently 8 sectors (4KB) for NTFS volumes,
    but accounting for this is arbitrary, especially since Disk Management uses MB
